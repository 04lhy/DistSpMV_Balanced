## 4. 可复现性说明

### 基本假设
1. **对称重排序：** 行与列使用相同的置换。
2. **连续行划分：** 每个进程获得连续的一块行。
3. **列归属 = 初始行分布：** 列 `c` 归属于进程 `p`，其中 `r_start_p ≤ c < r_end_p`。
4. **方阵：** 算法假设 `nrows == ncols`，用于划分逻辑。
5. **IEEE 双精度：** 所有浮点计算使用 `float64`。

### 与论文的已知差异
1. **METIS vs. RCM：** 当 METIS 不可用时，使用 RCM 进行重排序。RCM 能够减少带宽，但不会直接针对进程数进行优化，可能导致通信量高于论文报告的结果。
2. **Python GIL：** 多线程本地 SpMV 使用 Python 线程，受 GIL 限制。算法结构（按非零元素数均衡划分行、私有累加器）是正确的，但要获得实际的线程加速效果，需要使用 C/C++ 或 Numba JIT 后端。
3. **Alltoallv vs Isend/Irecv：** 论文可能使用 `MPI_Alltoallv` 进行向量交换。本实现使用点对点的 `Isend/Irecv`，语义上等价，但在某些互联网络上的延迟特性可能不同。

### 调优建议
1. **线程亲和性：** 设置 `OMP_PROC_BIND=close` 和 `OMP_PLACES=cores`。
2. **NUMA 感知：** 在多路系统中使用 `numactl --cpunodebind=<N> --membind=<N>`。
3. **MPI 急切发送阈值：** 调整 `OMPI_MCA_btl_vader_eager_limit` 以优化小消息延迟。
4. **进程到核心的映射：** 在 OpenMPI 中使用 `--map-by socket:PE=$OMP_NUM_THREADS`。
